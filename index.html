🔧 完整功能分析表

九大功能模組的詳細技術規劃與實作策略

功能分類 子功能 說明 技術需求
1️⃣ 課文生成 使用者輸入條件生成教材 根據「程度、主題、母語」等，生成文章/對話 前端表單、後端轉送 API 請求（OpenAI）
2️⃣ 課文改寫 將現有教材簡化重構 貼上原文，系統改寫為適合不同程度的版本 同上，前端傳入原文，後端呼叫 ChatGPT
3️⃣ 學生背景設定 可選程度、語言、字數、主題 控制生成教材的參數 動態表單設計 + 請求 JSON 建構
4️⃣ 預覽與修改 教師可即時預覽並自行微調 類似「生成初稿 → 教師微調 → 完稿」 即時結果區塊、複製/編輯功能
5️⃣ 題目生成 自動生成問答題或討論題 根據教材生成理解題，提升互動性 二階段 prompt or plugin 模組
6️⃣ 共用教案庫 教師可上傳、查看、分享改寫成果 教材管理、標籤、分類搜尋 資料庫（MongoDB/Firebase）、帳號系統
7️⃣ 使用者登入 教師帳號登入、儲存歷史紀錄 儲存個人設定與教案 JWT 驗證、OAuth2、資料綁定
8️⃣ 多語界面 繁中／簡中／英文切換 介面可切換語系 i18n 套件（如 i18next）
9️⃣ 後台統計 查看使用量、主題熱門趨勢 協助分析哪些教材最受歡迎 Log + Dashboard（如 Chart.js + Admin UI）

六大核心功能模組

AI 模型策略 (Prompt Engineering)

模型選擇
  • GPT-3.5 / GPT-4 文本生成引擎
  • 依據教材風格設計 prompt
  • System 設定角色為華語教師
提示策略
  • 多段提示：內容生成 + 問題生成
  • 模板類型：語氣、程度、風格控制
  • 語料校正：詞彙頻率與語法適切性
